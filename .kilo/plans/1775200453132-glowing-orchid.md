# 视频生成功能实现计划

## 概述

在现有图像生成功能（`image_generation`）的架构基础上，新增视频生成功能模块 `video_generation`。支持以下触发方式：
1. `/video` Discord 斜杠命令（直接触发）
2. AI 工具调用（通过 `generate_video` 工具函数自动触发）

同时支持两种生成模式：
- **文生视频**（text-to-video）：文字描述 → 视频
- **图生视频**（image-to-video）：上传图片 → 视频

首批集成 **OpenAI 兼容端口**，后续可扩展 Kling AI、Google Veo 等。

---

## 难度评估

| 维度 | 评分 | 说明 |
|---|---|---|
| 整体难度 | ★★★☆☆ 中等 | 架构完全参考图像生成，但视频生成有独特挑战 |
| API 集成 | ★★☆☆☆ 较低 | OpenAI 兼容端口格式统一，直接复用现有 httpx 调用模式 |
| 异步轮询 | ★★★☆☆ 中等 | 视频生成耗时长（10s-3min），需要任务轮询机制，超时处理更复杂 |
| Discord 交互 | ★★★★☆ 较高 | 视频文件大（可能几十 MB），Discord 文件上传有 8MB/25MB 限制；图生视频需要处理用户上传图片 |
| AI 工具集成 | ★★★☆☆ 中等 | 需处理"视频结果无法直接嵌入多模态消息"的限制，返回视频链接或文件 |

**主要技术挑战：**
1. **视频文件大小**：Discord 普通用户限制 8MB，Nitro 25MB/500MB。视频可能超限，需提供备选方案（链接/临时 URL）
2. **生成时间长**：需异步任务 + 状态推送（进度消息更新）
3. **图生视频的附件处理**：Discord 附件 URL 需先下载再传给 API

## 详细实现步骤

### 步骤 1：配置模块 `video_generation_config.py`

从环境变量读取所有配置，结构参考 `image_generation_config.py`：

```python
VIDEO_GENERATION_CONFIG = {
    "ENABLED": os.getenv("VIDEO_GENERATION_ENABLED", "False").lower() == "true",
    "SLASH_COMMAND_ENABLED": os.getenv("VIDEO_GENERATION_SLASH_COMMAND_ENABLED", "True").lower() == "true",
    "TOOL_CALL_ENABLED": os.getenv("VIDEO_GENERATION_TOOL_CALL_ENABLED", "True").lower() == "true",
    "DEFAULT_PROVIDER": os.getenv("VIDEO_GENERATION_DEFAULT_PROVIDER", "openai_compatible"),
    "TIMEOUT_SECONDS": int(os.getenv("VIDEO_GENERATION_TIMEOUT_SECONDS", "300")),
    "POLL_INTERVAL_SECONDS": float(os.getenv("VIDEO_GENERATION_POLL_INTERVAL_SECONDS", "5")),
    "MAX_PROMPT_LENGTH": int(os.getenv("VIDEO_GENERATION_MAX_PROMPT_LENGTH", "1000")),
    "COIN_COST": int(os.getenv("VIDEO_GENERATION_COIN_COST", "50")),
    "OPENAI_COMPATIBLE": {
        "API_KEY": os.getenv("VIDEO_OPENAI_COMPATIBLE_API_KEY"),
        "BASE_URL": os.getenv("VIDEO_OPENAI_COMPATIBLE_BASE_URL", ""),
        "T2V_MODEL": os.getenv("VIDEO_T2V_MODEL", ""),   # text-to-video 模型
        "I2V_MODEL": os.getenv("VIDEO_I2V_MODEL", ""),   # image-to-video 模型
    },
    # 后续可扩展 KLING / VEO 等配置块
}
```

**开发者控制开关**：
- `VIDEO_GENERATION_ENABLED`：总开关（False = 整个功能不可用）
- `VIDEO_GENERATION_SLASH_COMMAND_ENABLED`：单独控制 `/video` 命令
- `VIDEO_GENERATION_TOOL_CALL_ENABLED`：单独控制 AI 工具调用

### 步骤 2：数据模型 `providers/base.py`

```python
@dataclass(slots=True)
class VideoGenerationRequest:
    prompt: str
    mode: str = "text"              # "text" | "image"
    image_bytes: Optional[bytes] = None
    image_mime_type: str = "image/jpeg"
    aspect_ratio: str = "16:9"
    duration_seconds: Optional[int] = None
    provider: Optional[str] = None

@dataclass(slots=True)
class VideoGenerationResult:
    success: bool
    provider: str
    model: Optional[str] = None
    video_bytes: Optional[bytes] = None   # 视频内容（可能为 None，用 video_url 代替）
    video_url: Optional[str] = None       # 临时视频链接
    mime_type: str = "video/mp4"
    duration_seconds: Optional[float] = None
    metadata: Dict[str, Any] = field(default_factory=dict)
    error_code: Optional[str] = None
    error_message: Optional[str] = None

class VideoGenerationProvider(Protocol):
    async def generate(self, request: VideoGenerationRequest) -> VideoGenerationResult: ...
```

### 步骤 3：OpenAI 兼容 Provider `openai_compatible_video_provider.py`

OpenAI 的视频生成 API 通常是**异步任务模式**（提交任务 → 轮询状态 → 获取结果），各兼容服务商格式略有差异，实现策略：

**方案 A（推荐）- 先支持 OpenAI 视频 API 格式：**
```
POST /v1/videos/generations
  Body: { model, prompt, size/aspect_ratio, duration, ... }
  
Response: { id: "task_xxx", status: "queued" }

GET /v1/videos/generations/{id}
  Response: { status: "succeeded", data: [{ url: "..." }] }
```

**关键逻辑：**
1. 提交生成任务，获取 `task_id`
2. 异步轮询（每 `POLL_INTERVAL_SECONDS` 秒轮询一次）
3. 达到 `TIMEOUT_SECONDS` 超时则返回失败
4. 成功后：若返回 `url`，尝试下载视频字节；若文件过大（>8MB），则只保留 URL
5. 图生视频：将 `image_bytes` base64 编码后作为 `image` 字段传入

### 步骤 4：服务层 `video_generation_service.py`

结构直接参考 `image_generation_service.py`：

```python
class VideoGenerationService:
    def __init__(self):
        self.providers: Dict[str, VideoGenerationProvider] = {
            "openai_compatible": OpenAICompatibleVideoProvider(),
            # 后续: "kling": KlingVideoProvider(),
        }
    
    async def generate(self, request: VideoGenerationRequest) -> VideoGenerationResult:
        if not VIDEO_GENERATION_CONFIG["ENABLED"]: ...
        provider_name = request.provider or VIDEO_GENERATION_CONFIG["DEFAULT_PROVIDER"]
        provider = self.providers.get(provider_name)
        return await provider.generate(request)

video_generation_service = VideoGenerationService()
```

### 步骤 5：Discord Cog `video_generation_cog.py`

```python
@app_commands.command(name="video", description="使用 AI 生成一段视频")
@app_commands.describe(
    prompt="视频内容描述",
    image="上传图片以生成图生视频（可选）",
    aspect_ratio="画面比例（16:9 / 9:16 / 1:1）",
)
async def video(self, interaction, prompt: str, image: Optional[discord.Attachment] = None, aspect_ratio: str = "16:9"):
    ...
```

**交互流程：**
1. 检查功能开关（`SLASH_COMMAND_ENABLED`）
2. 检查类脑币余额，扣费
3. `interaction.response.defer(ephemeral=True, thinking=True)`
4. 若有图片附件，下载图片字节
5. 发送进度消息（"正在生成视频，预计需要约 X 秒..."）
6. 调用 `video_generation_service.generate(request)`
7. 结果处理：
   - 若有 `video_bytes` 且大小 ≤ Discord 限制：作为 `discord.File` 发送
   - 若视频过大或只有 `video_url`：发送链接
   - 失败则退款
8. 扣费使用现有 `coin_service`，reason 字段写 `"AI 视频生成"`

### 步骤 6：AI 工具函数 `generate_video.py`

参考 `generate_image.py` 结构，但需处理视频无法直接嵌入多模态消息的问题：

```python
@tool_metadata(
    name="视频生成",
    description="根据用户明确的视频生成请求生成一段视频。仅在用户明确要求生成视频时调用。",
    emoji="🎬",
    category="创作",
)
async def generate_video(params: GenerateVideoParams | str | None = None, ...) -> dict:
    ...
    result = await video_generation_service.generate(request)
    if not result.success:
        return {"success": False, "error_message": ...}
    
    return {
        "success": True,
        "provider": result.provider,
        "model": result.model,
        # 视频数据返回方式：
        "video_data": {
            "data": result.video_bytes,      # 可能为 None
            "mime_type": result.mime_type,
            "url": result.video_url,          # 视频链接（可能为 None）
        },
        "message": "已根据请求生成视频。",
    }
```

**工具调用的视频结果处理**（需修改现有工具执行器）：
- 返回 `video_data.data` 时，如同图片一样作为 Discord 文件附件发送
- 返回 `video_data.url` 时，在消息中附带链接
- 需要检查现有 `ai_chat_cog.py` / 工具执行器如何处理图片结果，并在相同位置添加视频结果处理逻辑
